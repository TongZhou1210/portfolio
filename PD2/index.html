3) Technical Structure

The codebase follows a modular, separation-of-concerns approach: each file has a focused responsibility, and UI code is kept separate from pure game logic.

  • dice.js ‚Äî Represents the set of dice, including current values, ‚Äúheld‚Äù flags, and random roll logic.
  • yatzyEngine.js ‚Äî Contains all Yatzy scoring rules and helper functions for calculating category scores and totals.
  • yatzyGame.js ‚Äî Coordinates the turn flow (rolls left, which categories are available) and communicates with the engine to update the score.
  • script.js ‚Äî Connects the HTML page to the game logic: attaches event listeners, updates the DOM, and renders the current state.
  • styles.css ‚Äî Defines the visual design: colour palette, typography, layout, and responsive behaviour.

The game uses a one-way data flow: user actions trigger game-logic functions, the game state is updated, and then a render function refreshes the DOM based on that state. This makes the behaviour easier to reason about, extend, and test.
